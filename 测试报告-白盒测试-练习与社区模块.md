# 《测试报告》白盒测试部分

## 1. 测试概述

本次白盒测试面向 `RAG-Knowledge-Assistant` 项目的练习管理模块与课程社区模块，测试对象为成员 C 负责的后端代码。测试工作围绕 `ExerciseController`、`PostController` 及其关联的 `service`、`repository` 层展开，重点验证练习题查看与维护、课程社区发帖、评论、回复等业务逻辑在正常输入、异常输入和缺陷触发输入下的执行情况。

本轮测试已完成白盒测试设计、测试用例构造、测试代码编写、自动化执行以及覆盖率统计分析。测试过程中共执行 26 个自动化测试用例，其中 23 个通过，3 个失败。失败用例揭示了当前系统在空题目集校验、评论目标存在性校验和回复目标存在性校验方面存在缺陷。

## 6. 缺陷与问题分析

本轮白盒测试共发现 3 个缺陷，具体如下：

### 缺陷 1：系统允许上传空题目集

1. 对应测试用例：`test_upload_empty_question_list_should_be_rejected`
2. 实际结果：接口返回 `200 OK`
3. 预期结果：应返回 `400` 或 `422`，拒绝空题目集
4. 原因分析：
   - 当前 `ExerciseController.upload_exercise_set()` 和 `ExerciseService.create_exercise_set()` 仅对请求进行接收和保存，没有对 `questions` 是否为空进行校验。
   - 导致系统可以创建没有任何题目的习题集，不符合业务常识，也会影响前端练习页显示和后续统计逻辑。

### 缺陷 2：系统允许对不存在的帖子发表评论

1. 对应测试用例：`test_add_comment_should_reject_missing_post`
2. 实际结果：接口返回 `200 OK`
3. 预期结果：应返回 `400` 或 `404`，拒绝对不存在的帖子发表评论
4. 原因分析：
   - 当前 `PostController.add_comment()` 在调用 `create_comment()` 前未验证 `post_id` 是否存在。
   - 评论逻辑直接将 `post_id` 传入仓储层保存，缺少业务对象存在性校验。
   - 这会导致非法评论数据被接受，影响社区数据一致性。

### 缺陷 3：系统允许回复不存在的父评论

1. 对应测试用例：`test_add_comment_should_reject_missing_parent`
2. 实际结果：接口返回 `200 OK`
3. 预期结果：应返回 `400` 或 `404`，拒绝回复不存在的父评论
4. 原因分析：
   - 当前 `PostController.add_comment()` 未验证 `parent_id` 是否对应真实评论记录。
   - 仓储层 `add_comment()` 直接保存 `parent_id`，缺乏回复链路合法性校验。
   - 这会导致评论树出现断裂，影响后续评论展示与回复结构维护。

## 7. 测试结论

从白盒测试结果来看，练习管理模块与课程社区模块的主要业务逻辑已经可以正常运行，控制器、服务层和仓储层之间的数据传递、分页查询、权限控制、匿名映射、评论回复等核心路径均已得到有效验证。

同时，测试也发现了 3 个明确缺陷，说明当前系统在输入合法性校验和业务对象存在性校验方面仍有改进空间。虽然这些缺陷不影响大部分正常功能路径的执行，但会对系统的数据完整性和业务约束造成风险。因此，在后续修复过程中，应优先补充以下校验逻辑：

1. 习题集上传时校验题目列表不能为空
2. 评论前校验帖子是否存在
3. 回复前校验父评论是否存在

总体来看，本人负责模块的白盒测试已经达到课程作业对“测试设计、测试实现、测试执行、缺陷发现”的基本要求，可作为小组总测试报告中白盒测试部分的重要组成内容。
