## 二、采购该配置的理由

### 1. 实例规格：ac8.xlarge.4

在“课程智能助手系统”中，教师上传课程资料（PDF/PPT/Word）后，系统需 **并行多模态解析与向量化嵌入**，对 CPU 性能和内存访问速度要求较高，并且需要反复调用大模型 API 。该实例可在稳定提供高计算性能的同时，保证出色的网络吞吐，满足系统的高并发、实时响应与大规模数据交互需求。

---

### 2. CPU：4 vCPUs

系统采用 **Celery 异步任务队列** 执行后台任务（文档解析、向量化计算、知识图谱更新等）。
每个 Celery Worker 通常占用一个 CPU 核心，因此 4 核配置可同时运行多个 Worker 实例，从而：

* 提升任务并发度与系统响应速度；
* 避免任务阻塞与队列积压；
* 保证多用户并行处理时系统依旧流畅稳定。

对于中等规模 RAG 系统部署而言，4 vCPUs 在性能与成本间达成理想平衡，既能保证实时性，又可支撑多进程并行工作。

---

### 3. 内存：16 GiB

在项目中，FAISS 或 Chroma 向量数据库需常驻内存以保持高检索速度；
同时，知识图谱可视化、会话缓存、任务队列（如 Celery）均需占用内存资源。

8 GiB 内存可同时满足以下需求：

* RAG 检索缓存 + Web 服务常驻进程；
* Neo4j 图谱节点与边查询缓存；
* Celery Worker 执行任务时的模型与依赖占用。

经测试，8 GiB 会频繁因为 `out of memory` 使得进程被 kill。

### 4. 网络带宽：最高 12.5 Gbit/s

课程智能助手系统的核心交互（学生问答、教师资料上传）依赖 WebSocket 与 REST 接口，此外还会频繁调用外部大语言模型（DeepSeek、QWen）API。

选择最大带宽的意义在于：

* 保证与外部 LLM 服务之间的 **高速出网带宽**，避免生成延迟；
* 上传/下载多模态课程资料（PDF、PPT、图片、音频）时速度更快。

高带宽结合按流量计费模式，可在低访问时降低成本，在高峰期保持体验流畅。

---

### 5. 操作系统：Ubuntu Server 22.04

Ubuntu 22.04 LTS 是当前最稳定、兼容性最强的主流 Linux 服务器系统之一。

优势包括：

* 官方长期支持，安全更新稳定；
* 原生支持 Python 3.10+、Node.js、Docker、FAISS、Neo4j、LangChain、Celery 等组件；
* 与项目现有开发环境完全一致，方便直接迁移依赖环境；
* 便于自动化部署（CI/CD）与容器化管理。

该系统在性能、兼容性与生态支持方面均能满足课程智能助手系统的长期运行需求。

---

### 6. 计费方式：按需计费（按流量计费，带宽选择最大）

由于当前系统处于 **研发与测试阶段**，访问频率和并发量尚不稳定。按需计费模式可：

* 在使用时计费，避免包月浪费；
* 按实际外网出流量计费，成本极低（单次问答数据量仅数百 KB）；
* 支持随时升级或释放实例，方便灵活扩容；
* 结合最大带宽，在高峰期维持快速访问体验。
