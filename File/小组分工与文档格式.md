# 小组分工与文档格式

## 一、三人分工

### 成员 A：黑盒测试负责人

负责黑盒测试全部相关工作，独立完成黑盒测试链路的设计、执行与整理。

具体任务：

1. 基于《需求说明文档》确定测试范围、功能点和核心流程  
2. 编写《测试计划》（明确方法：等价类 / 边界值 / 决策表）  
3. 设计黑盒测试用例  
4. 执行测试并记录结果  
5. 整理测试执行记录或测试代码  
6. 编写《测试报告》

负责的黑盒功能点：

1. 用户注册
2. 用户登录
3. 个人信息查看
4. 个人信息修改
5. 教师创建课程
6. 学生加入课程
7. 学生退出课程
8. 课程资料上传
9. 教师解散课程
10. 课程资料查看
11. 课程资料下载
12. 课程资料 PDF 预览
13. 课程资料删除
14. 练习题上传
15. 练习题查看
16. 练习题作答
17. 作答结果反馈
18. 课程社区发帖
19. 课程社区评论
20. 课程社区回复
21. 核心流程黑盒验证

核心流程可定义为：

教师登录 -> 创建课程 -> 上传资料 -> 学生登录 -> 加入课程 -> 查看资料 -> 完成练习 -> 参与课程社区互动

### 成员 B：白盒测试负责人一

负责基础业务模块与资料管理模块的白盒测试工作。

具体任务：

1. 根据代码结构完成本人负责模块的白盒测试设计。
2. 采用代码覆盖、基本路径覆盖和数据流分析等方法进行测试。
3. 编写并执行本人负责模块的白盒测试代码。
4. 统计并分析本人负责模块的覆盖率结果。
5. 完成本人负责部分的《测试计划》白盒内容。
6. 完成本人负责部分的《测试报告》白盒内容。

负责模块：

1. `用户与认证模块（LoginController、UserController）`
2. `课程管理模块（CourseController）`
3. `课程资料管理模块（rag_controller）`

负责内容：

1. 登录认证逻辑
2. 用户信息处理逻辑
3. 课程创建、加入、退出、解散等核心业务逻辑
4. 资料上传、删除、下载、预览相关逻辑

### 成员 C：白盒测试负责人二

负责练习与社区模块的白盒测试工作。

具体任务：

1. 根据代码结构完成本人负责模块的白盒测试设计。
2. 采用代码覆盖、基本路径覆盖和数据流分析等方法进行测试。
3. 编写并执行本人负责模块的白盒测试代码。
4. 统计并分析本人负责模块的覆盖率结果，并协助整体覆盖率达到 95% 以上。
5. 完成本人负责部分的《测试计划》白盒内容。
6. 完成本人负责部分的《测试报告》白盒内容。
7. 完成未来扩展适配部分，提出 2 到 3 个未来功能变更并给出对应的黑盒、白盒测试调整方案。

负责模块：

1. `练习管理模块（ExerciseController）`
2. `课程社区模块（PostController）`

负责内容：

1. 练习题查看、作答、反馈相关逻辑
2. 社区发帖、评论、回复相关逻辑

## 二、统一格式

### 《测试计划》统一格式

1. 测试目标
2. 测试范围
3. 测试依据
4. 测试环境与工具
5. 测试方法
6. 被测对象说明
7. 测试执行安排
8. 风险与说明

说明：

黑盒测试第 6 节写“功能点与核心流程”。

白盒测试第 6 节写“被测代码模块与覆盖目标”。

### 《测试报告》统一格式

1. 测试概述
2. 测试环境与工具
3. 测试对象说明
4. 测试方法说明
5. 测试执行结果
6. 缺陷与问题分析
7. 测试结论

白盒测试报告增加：

8. 覆盖率结果与分析
